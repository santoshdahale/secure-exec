## Codebase Patterns
- Use `pnpm turbo run check-types` for TypeScript typecheck (not `typecheck`)
- Rust crate lives at `crates/v8-runtime/`; binary name is `secure-exec-v8`
- `cargo check` in `crates/v8-runtime/` validates Rust compilation
- rusty_v8 crate is imported as `v8` (e.g., `v8 = "130"` in Cargo.toml)
- IPC enums use `#[serde(tag = "type")]` (internally tagged) for JS interop
- Use `serde_bytes` for `Vec<u8>` fields to get MessagePack bin format
- Use `rmp_serde::to_vec_named` (not `to_vec`) for named-field serialization
- Framing functions: `write_message(&mut writer, &msg)` and `read_message(&mut reader)` in `ipc.rs`
- Max IPC message size is 64MB (`MAX_MESSAGE_SIZE` constant in `ipc.rs`)
- V8 tests MUST run with `--test-threads=1` to avoid SIGSEGV from parallel V8 operations
- `signal-hook::flag::register` clears the AtomicBool to `false` on signal receipt
- `v8::Context::new(scope, Default::default())` — v130 requires a `ContextOptions` second arg
- Use `v8::Global<v8::Context>` to persist context handles across HandleScope lifetimes
- Consolidate V8 tests into a single `#[test]` fn to avoid inter-test isolate state issues
- V8 platform MUST be initialized on the main thread before spawning session threads (SIGSEGV otherwise)
- Session tests use `#[cfg(not(test))]` to skip V8 isolate creation; V8 lifecycle tested in `isolate::tests`
- `session_id` is `String` in IPC types (not u32/u128) for JS interop; use hex nonces for 128-bit security
- Session concurrency is configurable via `SECURE_EXEC_V8_MAX_SESSIONS` env var (default: available CPUs)
- Use `std::thread::available_parallelism()` instead of `num_cpus` crate for CPU count

  - Rust's `UnixListener::bind()` combines bind() and listen() — can't separate them without raw libc
